# BigQuery Release Notes Dashboard & X/Twitter Sharer

A beautiful, high-fidelity developer dashboard that aggregates Google Cloud BigQuery release notes, parses individual updates, and facilitates editing and sharing specific notes directly to X (formerly Twitter).

---

## 🚀 Key Features

* **Sub-Entry Splitting**: Daily release notes inside Google's Atom feed are parsed using `BeautifulSoup` and split by `<h3>` headings into individual card objects. Users can filter, search, and Tweet specific updates (rather than the entire day's notes).
* **Smart Local Caching**: Includes a file-based JSON cache system. Responses load instantly from the cache, while a **Refresh** button allows live sync with Google's XML feed on demand.
* **X / Twitter Composer**: Opens a native HTML5 `<dialog>` compose window. Text templates are dynamically formatted and auto-truncated to fit within Twitter's 280-character limit.
* **SVG Character Counter**: Includes a custom SVG circular progress ring that shifts colors from primary blue to warning amber, and finally to alert red as users type and approach/exceed the character limit.
* **Interactive Filters**: Quick filter buttons for different categories (Features, Issues, Resolved, and Deprecations), live keyphrase search, and chronological sorting.
* **Premium Dark Mode**: Built with custom HSL color palettes, responsive flex/grid layouts, ambient glassmorphic blurred backdrops, and custom styled scrollbars.

---

## 🛠️ Tech Stack

* **Backend**: Python Flask, `feedparser` (Atom parsing), `BeautifulSoup4` (HTML fragmentation/segmentation), `urllib` (HTTP requests)
* **Frontend**: Vanilla HTML5, CSS3, and JavaScript
* **API Framework**: REST JSON API
* **Compatibility**: Native `<dialog>` and Invoker commands with conditional polyfills loaded via CDN for legacy support.

## ⚙️ Setup & Local Running

### Prerequisites
Make sure you have Python 3.10+ installed.

### 1. Initialize Virtual Environment & Install Dependencies
Navigate to the root directory and set up Python packages:

```bash
# Create virtual environment
python3 -m venv venv

# Activate virtual environment
source venv/bin/activate

# Install dependencies
pip install Flask requests beautifulsoup4 feedparser
```

### 2. Run the Development Server
Start the Flask application:

```bash
python app.py
```
By default, the server runs in debug mode on **[http://127.0.0.1:5000](http://127.0.0.1:5000)**.

### 3. Build & Cache Details
* The cache automatically refreshes after 1 hour (`3600` seconds).
* Clicking the **Refresh** button in the top right corner triggers `/api/releases?refresh=true` which bypasses the cache, pulls fresh data from Google Cloud, and rewrites `releases_cache.json`.
